# v1.1.1 - Skill Structure Improvement Batch

**Date:** 2026-04-12

This release is a targeted quality pass over 10 skills identified by the improve-skills batch plan.

The main work was structural rather than functional: compressing one oversized internal skill, adding realistic gotchas and examples to under-specified skills, and standardizing Impact Report coverage across the `secure-*` security skills touched by the batch.

---

## Changed

- `agent-creator` was compressed from 262 lines to 138 lines by consolidating topology instructions, simplifying failure handling, and removing duplicated example structure.
- `agent-system-architecture` now includes domain-specific gotchas and a realistic concurrent PR-review architecture example.
- `architectural-decision-log` now has a narrower role definition plus ADR-specific gotchas and a full SQLite vs PostgreSQL example.
- `generate-changelog` now has stronger changelog authoring constraints, domain-specific gotchas, and a concrete release-note example.
- `technical-debt-audit` now prioritizes blast radius over TODO count and includes a realistic audit example tied to this repo.
- `implementation-plan` now has stronger planning gotchas and a full implementation-plan example with phases, DoD, and risk handling.
- `test-driven-development` now encodes stricter Red-Green-Refactor failure-mode guidance and includes a complete TDD walkthrough example.

## Security Skill Improvements

- `secure-skill-content-sanitization` now ends with the standard `Impact Report` block required by the repo rubric.
- `secure-skill-repo-ingestion` now ends with the standard `Impact Report` block required by the repo rubric.
- `secure-skill-runtime` now ends with a compact `Impact Report` block while staying under the tighter security-skill line limit.
